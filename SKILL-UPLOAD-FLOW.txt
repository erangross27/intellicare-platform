┌─────────────────────────────────────────────────────────────────────────────┐
│                    WHAT HAPPENS UNDER THE HOOD                              │
└─────────────────────────────────────────────────────────────────────────────┘

STEP 1: YOUR APP SENDS REQUEST
───────────────────────────────
{
  "message": "Show Helen's allergies",
  "skills": [
    "intellicare-search-patients-by-name",
    "intellicare-get-allergies"
  ]
}

STEP 2: CLAUDE API RECEIVES IT
──────────────────────────────
"OK, I see skills: search-patients, get-allergies

Let me look them up in my storage..."

  Database Query:
  SELECT * FROM skills
  WHERE id IN ('intellicare-search-patients-by-name',
               'intellicare-get-allergies')

  Results returned INSTANTLY (skills already stored)

STEP 3: CLAUDE LOADS SKILL DEFINITIONS
───────────────────────────────────────
searchPatientsByName {
  description: "Search for patients by name",
  parameters: {
    firstName: string,
    lastName: string
  }
}

getAllergies {
  description: "Get allergies grouped by severity",
  parameters: {
    patientId: string
  }
}

STEP 4: CLAUDE DECIDES WHICH TO USE
───────────────────────────────────
"Message: 'Show Helen's allergies'

 I have skills available:
 ✓ searchPatientsByName - can find Helen
 ✓ getAllergies - can get allergies

 I'll use both:
 1. First search for Helen
 2. Then get her allergies"

STEP 5: CLAUDE EXECUTES SKILLS
───────────────────────────────
CALL 1: searchPatientsByName({
  firstName: "Helen",
  lastName: "Cox"  // implied from context
})
RETURNS: { patientId: "12345" }

CALL 2: getAllergies({
  patientId: "12345"  // from result above
})
RETURNS: [
  { allergen: "Penicillin", severity: "severe" },
  { allergen: "Shellfish", severity: "moderate" }
]

STEP 6: CLAUDE COMPOSES RESPONSE
────────────────────────────────
"Based on the skill results:

Helen Cox has 2 known allergies:
- Penicillin (SEVERE)
- Shellfish (MODERATE)"

STEP 7: RESPONSE SENT TO USER
──────────────────────────────
✅ Result displayed in chat

                    KEY INSIGHT: REFERENCE vs DEFINITION

    Current (WITHOUT Skills):
    ├─ Every request includes full function code
    ├─ 5,000+ tokens of definitions per call
    └─ Wasteful! (Same definitions sent repeatedly)

    New (WITH Skills):
    ├─ Skills stored once on Claude's server
    ├─ Every request just references by name (~50 tokens)
    └─ Efficient! (Definitions loaded from storage)
