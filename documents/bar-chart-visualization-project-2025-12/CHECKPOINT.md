# Bar Chart Visualization Project - December 2025

## Project Overview
Add horizontal bar chart visualization to medical templates with numeric score fields.
Following the pattern established in `neuropsychological_assessments` template (commit 33d74a93).

## Reference Implementation
- **Template:** neuropsychological_assessments
- **Commit:** 33d74a93
- **Pattern Memory ID:** 693426701ca57d1d25ef0796

## Implementation Pattern
1. Add `extractScore()` function - parses %, percentile, T-score, SS, scaled scores
2. Add `getScoreColor()` function - Green (80+), Blue (60-79), Orange (40-59), Red (<40)
3. Add `BarChart` component - horizontal bar with label, fill, percentage
4. Add `Legend` component - color key for score ranges
5. Add CSS styles for bar charts
6. Insert chart at TOP of relevant section, keep existing content below

## Additional Requirements (December 7, 2025 - clinical_scores lessons)
7. Add `safeString()` helper - prevents React crash on empty objects `{}`
8. Add `splitByComma()` / `splitByPeriod()` - for text splitting in mini-cards
9. Add `filteredChartData` - filters bars based on search term (not just highlights!)
10. Row-level copy buttons - each split item gets its own `numbered-row` + `copy-btn`
11. Blue-only theme - transparent buttons with `#2563eb` border, no purple gradient
12. PDF `splitCommaSeparatedText()` - for numbered list display

## Additional Requirements (December 7, 2025 - fall_risk_assessments lessons)
13. `filteredChartData` MUST use `.every()` (AND logic), NOT `.some()` (OR logic)
    - Bug: "Timed Up and Go" search showed Chair Stand because "and" matched "St-and-"
    - Fix: `searchWords.every(word => combinedText.includes(word))`
14. Single-char remnant filtering for parentheses search (e.g., "(T" → "t" ignored)
15. PDF must use `allPdfRecords` (original records), NOT `filteredRecords`

## Lessons Learned (December 7, 2025)

### Mini-Card CSS Text Alignment
**Problem:** Text in mini-cards appears centered instead of left-aligned
**Fix:** Use `align-items: flex-start` (not center) on flex containers + `text-align: left` on content-value
**Memory ID:** 693590f7a118974dca8ee180

### PDF Page 1 Empty Issue
**Problem:** `wrap={false}` on record container pushes all content to page 2
**Fix:** Put `wrap={false}` on EACH SECTION instead of the record container
**Memory ID:** 693590f7a118974dca8ee181

### Qualitative Values in Bar Charts (ClockDrawing Pattern)
**Problem:** Some assessments have qualitative values (Normal/Abnormal) not numeric scores
**Fix:** Handle with colored badge instead of progress bar - mark as `type: 'qualitative'` in config
**Memory ID:** 6935a519a118974dca8ee183

### Bar Chart Category Groupings
**Problem:** Flat list of many scores is overwhelming
**Fix:** Group into medical categories (Cognitive → Functional → Mobility → Nutrition → Mental Health) with visual headers
**Memory ID:** 6935a519a118974dca8ee184

### Section Order - Context Before Data
**Problem:** Clinicians need context (WHO/WHEN/WHERE) before interpreting data
**Fix:** Assessment Information section FIRST, then Score Overview bar charts
**Memory ID:** 6935a51aa118974dca8ee185

### Remove Duplicate Sections
**Problem:** Same data appears in both bar chart AND Test Results/Findings sections
**Fix:** Remove duplicate sections - bar chart visualization is superior (visual + interpretations)
**Memory ID:** 6935a51aa118974dca8ee186

### Bar Chart Category-Level Search Filtering (December 8, 2025)
**Problem:** Searching for a category name (e.g., "Risk Scores") didn't filter to just that category's bars
**Fix:** In `getFilteredChartData()`, detect category name searches BEFORE individual bar filtering:
```javascript
const survivalCategoryMatches = searchWords.every(word => 'survival response'.includes(word));
const riskCategoryMatches = searchWords.every(word => 'risk scores'.includes(word));

if (survivalCategoryMatches) return chartData.filter(c => c.category === 'Survival & Response');
if (riskCategoryMatches) return chartData.filter(c => c.category === 'Risk Scores');
```
Also add category names to `searchableText` in `filteredRecords` and use `highlightText()` on category headers.
**Commit:** 6dd2550c

### Allergen Categorization Pattern for IgE Testing (December 9, 2025)
**Problem:** Allergy skin testing has many allergens that should be grouped logically for readability
**Solution:** Create categorizeAllergen() function using pattern matching to group allergens:
```javascript
const categorizeAllergen = (allergen) => {
  const lowerAllergen = allergen.toLowerCase();
  if (lowerAllergen.includes('aspergillus') || lowerAllergen.includes('penicillium') ||
      lowerAllergen.includes('alternaria') || lowerAllergen.includes('mold')) {
    return 'Molds/Fungi';
  }
  if (lowerAllergen.includes('dust') || lowerAllergen.includes('mite') ||
      lowerAllergen.includes('cockroach') || lowerAllergen.includes('latex')) {
    return 'Environmental';
  }
  // ... more patterns
  return 'Other';
};
```
Then use groupByCategory() to group chart data and render with category headers in specific order.
Also add getAllergenDescription() for educational short descriptions next to each allergen.
**Template:** allergy_skin_testing
**Commits:** 4ed5b69a, c781c316, 6ee4e284

### Granular wrap={false} for Large Bar Chart Sections (December 12, 2025)
**Problem:** When a bar chart section has multiple categories (e.g., 6 categories with 15+ bars), applying `wrap={false}` to the ENTIRE section causes text overlap. The content is too large for one page, but react-pdf tries to keep it together anyway.
**Solution:** Apply `wrap={false}` GRANULARLY - not to the whole section, but to smaller logical units:
```jsx
<View style={chartStyles.chartSection}>
  {/* Title + Legend stay together */}
  <View wrap={false}>
    <Text style={chartStyles.chartTitle}>Risk Assessment Overview</Text>
    <PDFLegend />
  </View>

  {/* Each category can wrap independently - header stays with its bars */}
  {categoryData.map((category, catIdx) => (
    <View key={category.id} style={chartStyles.chartCategory} wrap={false}>
      <PDFCategoryHeader name={category.name} description={category.description} />
      {category.charts.map((chart, cIdx) => (
        <PDFBarChart key={chart.key} {...chart} />
      ))}
    </View>
  ))}
</View>
```
This allows page breaks BETWEEN categories while keeping related content together.
**Template:** anesthesia_records
**Memory ID:** 693baf074ec7f0ea8820a2bc
