================================================================================
                              SYSTEM ARCHITECTURE
================================================================================

CORE COMPONENTS:

1. DrugInformationService (/apps/backend-api/services/drugInformationService.js)
   - Integrates 8 OpenFDA APIs (drug, device, food, tobacco)
   - Manages local drug interaction database
   - Performs safety assessments
   - Validates prescriptions

2. MedicationSafetyChecker (/apps/backend-api/services/medicationSafetyChecker.js)
   - High-level safety orchestration
   - Combines drug interactions + allergy checking
   - Generates comprehensive safety reports
   - Integration with patient medication/allergy data

3. AllergyChecker (/apps/backend-api/services/allergyChecker.js)
   - 20+ major allergen categories tracked
   - 150+ drug class cross-sensitivity rules
   - Alternative medication suggestions
   - Evidence-based cross-reactivity rates

4. ExternalApiGateway (/apps/backend-api/services/externalApiGatewayService.js)
   - Unified API management for OpenFDA + 6 other providers
   - KMS-encrypted API key management
   - Rate limiting (240 req/min for OpenFDA)
   - Response caching with TTL
   - Circuit breaker pattern for resilience

5. REST API Endpoints (/apps/backend-api/routes/external.js)
   - 4 drug endpoints (search, safety-check, interaction-check, validate-prescription)
   - 3 device endpoints (search, recalls, adverse-events)
   - 3 food endpoints (enforcement, search, products)
   - Rate-limited with comprehensive validation

================================================================================
                          KEY CAPABILITIES
================================================================================

DRUG INTERACTION CHECKING:
  - Local MongoDB database: intellicare_drug_data/drug_interactions
  - O(1) lookup performance with compound (drugA, drugB) index
  - 4 severity levels: CONTRAINDICATED, MAJOR, MODERATE, MINOR
  - Checks all medication pairs: N medications = N(N-1)/2 pairs
  - Sorted results by severity (critical first)
  - Integration with patient's current medications + allergies

ADVERSE EVENT MONITORING:
  - FDA MedWatch database integration
  - Continuous hourly checking for new alerts
  - Safety scoring algorithm (0-100 scale)
  - Alert thresholds: >10 events in 30 days = ALERT
  - Class I recall automatic escalation to CRITICAL

MEDICATION VALIDATION:
  - NDC number validation (10-11 digit format)
  - FDA database drug existence check
  - Dosage format validation
  - Class I recall detection
  - Interaction checking against existing medications
  - Safety scoring and risk assessment

ALLERGY CROSS-SENSITIVITY:
  - Beta-Lactam cross-reactivity (5-10%)
  - Sulfa drug coverage (loop diuretics, thiazides)
  - NSAID class interactions (5-10% cross)
  - Opioid sensitivity patterns
  - Local anesthetic typing (amide vs ester)
  - Contrast media and latex coverage
  - 4 severity levels with management recommendations

PRESCRIPTION MONITORING:
  - Hourly automatic activation of pending prescriptions
  - Automatic expiration when endDate arrives
  - Dual database pattern (global metadata + practice PHI)
  - Multi-tenant isolation
  - Complete audit trail

================================================================================
                         SAMPLE WORKFLOW
================================================================================

WHEN PRESCRIBER ADDS NEW MEDICATION:

1. System retrieves patient's current medications from database
2. System retrieves patient's allergies from database
3. System builds complete medication list (current + new + document)
4. DrugInformationService checks all medication pairs
   - Example: 5 meds = 10 pairs checked against local database
   - Returns: All interactions sorted by severity
5. AllergyChecker verifies no direct allergies
6. AllergyChecker checks cross-sensitivity risks
7. MedicationSafetyChecker generates comprehensive report:
   - isSafe: boolean
   - errors: Critical issues (major interactions, direct allergies)
   - warnings: Cautions (moderate interactions, cross-sensitivity)
   - alternatives: Suggested medications if needed
   - summary: Human-readable conclusion
8. Report logged to audit_logs collection with user attribution
9. Result displayed to prescriber with clear guidance

EXAMPLE ALERT:
  Medication: Aspirin
  Safety: NOT SAFE
  
  Critical Issues:
  - MAJOR interaction with Warfarin
  - Increased bleeding risk
  - Management: Monitor INR closely, consider alternative
  - Source: FDA Drug Interaction Database
  
  Recommendation:
  - Consult with prescribing physician immediately
  - Consider alternative medications (e.g., acetaminophen)
  - Do not administer without review

COMMON ISSUES:

Issue: No interactions found
- Check: Is drug_interactions collection populated?
- Check: Are medication names normalized (lowercase)?
- Verify: Database index exists on (drugA, drugB)

Issue: Safety check fails
- Check: Is FDA API key in KMS?
- Check: API rate limit not exceeded
- Verify: Network connectivity to api.fda.gov

Issue: Allergy check missing cross-sensitivity
- Check: Both allergens in database
- Check: Drug names in cross-sensitivity mapping
- Verify: Knowledge base is up to date
